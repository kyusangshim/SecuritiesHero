📁 src/hooks/auth/
목적: 사용자 인증과 관련된 모든 로직을 관리하는 커스텀 훅(Hook)들을 모아놓은 폴더입니다. 회원가입, 로그인, 로그아웃, 인증 상태 유지 등 핵심적인 인증 기능을 담당합니다.

포함된 훅(Hooks):

useAuth.tsx: 전역적인 인증 상태(로그인 여부, 사용자 정보, 로딩 상태)를 관리하는 핵심 훅입니다. 로그인, 로그아웃, 회원가입 API 호출과 토큰 기반의 인증 상태 확인 로직을 포함합니다. 또한, 인증된 사용자만 접근 가능한 페이지를 만들기 위한 HOC(withAuth)와 useRequireAuth 훅을 제공합니다.

useLogin.ts: 로그인 페이지의 상태와 로직을 전담하는 훅입니다. 사용자가 입력하는 아이디/비밀번호 상태, 입력값 검증, 에러 메시지, 아이디 저장 기능 등을 관리합니다.

useRegister.ts: 회원가입 폼의 상태와 로직을 관리하는 훅입니다. 폼 데이터, 필드별 유효성 검사(정규식 포함), 에러 상태, 그리고 useAuth의 register 함수를 호출하여 실제 회원가입을 처리하는 로직을 담당합니다.

useOAuthSuccess.ts: 소셜 로그인(OAuth) 성공 후, 리다이렉트된 페이지에서 최종 인증을 처리하는 훅입니다. 백엔드에 토큰을 요청하고, 성공 시 발급받은 accessToken을 localStorage에 저장한 후 메인 페이지로 이동시킵니다.

index.ts: auth 폴더 내의 모든 훅을 export하여 다른 컴포넌트에서 쉽게 불러올 수 있도록 하는 진입점 파일입니다.

특징:

중앙화된 인증 관리: 인증 관련 로직이 여러 컴포넌트에 흩어지지 않고 hooks/auth 폴더 내에서 집중적으로 관리됩니다.

상태와 로직의 분리: useAuth가 전역 인증 상태를, useLogin과 useRegister가 각 페이지의 UI 상태와 로직을 분리하여 역할과 책임이 명확합니다.

토큰 기반 인증: localStorage를 사용하여 JWT(Access Token)를 저장하고, 이를 기반으로 사용자의 로그인 상태를 확인하여 지속적인 인증을 구현합니다.

선언적 라우트 보호: withAuth HOC나 useRequireAuth 훅을 사용해 특정 페이지에 대한 접근 제어를 간편하게 설정할 수 있습니다.
