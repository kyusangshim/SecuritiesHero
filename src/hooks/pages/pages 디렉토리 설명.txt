📁 src/hooks/pages/
목적: 특정 페이지 전체의 상태(State)와 비즈니스 로직, 이벤트 핸들러를 관리하는 커스텀 훅을 모아놓은 폴더입니다. 각 훅은 하나의 페이지와 1:1로 매칭되어 해당 페이지의 "두뇌" 역할을 담당합니다.

포함된 훅(Hooks):

useMainPage.ts: 메인 페이지의 UI 상태(사용자 메뉴, 사이드 메뉴 노출 여부)와 사용자 인터랙션(버튼 클릭, 페이지 이동, 로그아웃)을 처리하는 훅입니다. useAuth 훅을 사용하여 사용자 정보와 로그아웃 기능을 가져와 사용합니다.

useMyPage.ts: 마이페이지의 모든 기능과 데이터를 총괄하는 훅입니다. 페이지 접근 시 인증 상태를 확인하고, 사용자 정보, 프로젝트 목록, 캘린더의 모든 상태(현재 월, 선택된 날짜, 이벤트 목록)와 관련 기능(월 이동, 이벤트 추가/수정/삭제)을 모두 포함하고 있습니다.

index.ts: pages 폴더 내의 모든 훅을 export하여 다른 컴포넌트에서 쉽게 불러올 수 있도록 하는 진입점 파일입니다.

특징:

페이지 단위 로직 캡슐화: 각 페이지에 필요한 모든 상태와 함수가 하나의 훅에 캡슐화되어 있어, UI 컴포넌트는 오직 화면을 그리는 데만 집중할 수 있습니다.

관심사의 분리(SoC): 페이지의 '어떻게' 보이는지(View)와 '어떻게' 동작하는지(Logic)를 명확하게 분리하여 코드의 가독성과 유지보수성을 크게 향상시킵니다.

상태 및 로직의 조합: 다른 훅(useAuth 등)을 가져와 조합하여 페이지에 필요한 더 복잡한 기능을 구성합니다.

API 호출 관리: 페이지 진입에 필요한 데이터(사용자 정보 등)를 가져오는 API 호출 로직을 포함하여 컴포넌트의 생명주기와 연동합니다.
