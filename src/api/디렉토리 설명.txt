📁 src/api/
목적: 백엔드 서버와의 모든 통신(API 요청)을 중앙에서 관리하고 설정하는 역할을 합니다. 애플리케이션 전체에서 일관된 방식으로 서버와 데이터를 주고받을 수 있는 통로를 제공합니다.

포함된 파일 및 역할:

axios.ts: HTTP 통신 라이브러리인 Axios의 설정을 마친 인스턴스(instance)를 생성하고 내보내는 파일입니다. 이 인스턴스는 앱의 모든 API 요청에 사용되는 기본 클라이언트가 됩니다.

특징:

중앙화된 API 설정: 백엔드 서버의 기본 주소(baseURL: "http://localhost:8080")나 기본 헤더(Content-Type)등이 한곳에서 관리됩니다. 서버 주소가 변경되어도 이 파일만 수정하면 되므로 유지보수가 매우 편리합니다.

자동화된 인증 토큰 주입: **요청 인터셉터(request interceptor)**를 사용하여, API를 요청할 때마다 localStorage에서 accessToken을 자동으로 읽어와 Authorization 헤더에 Bearer 토큰을 담아줍니다. 덕분에 각 API 호출 함수에서 매번 토큰을 추가하는 번거로운 작업을 할 필요가 없습니다.

쿠키 기반 통신 지원: withCredentials: true 옵션을 통해, API 요청 시 쿠키 정보를 함께 보내도록 설정되어 있습니다. 이는 (리프레시 토큰과 같은) 보안 관련 쿠키를 서버와 주고받기 위해 필수적인 설정입니다.
