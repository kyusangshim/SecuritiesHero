📁 src/pages/
역할: 애플리케이션의 각 페이지(라우트)에 해당하는 최상위 컴포넌트들을 모아놓은 폴더입니다. components 폴더의 UI 조각들과 hooks 폴더의 로직을 결합하여 하나의 완전한 페이지를 구성합니다.

포함된 페이지 및 설명:

LoginPage.tsx

역할: 사용자가 아이디/비밀번호나 소셜 계정으로 로그인하는 페이지의 전체 UI를 구성합니다.

기능: useLogin 훅에서 가져온 상태와 함수들을 LoginFormComponent에 연결하여 실제 로그인 기능을 수행하게 합니다.

특징: 페이지 자체에는 로직이 거의 없으며, 모든 동작은 useLogin 훅에 위임하는 구조입니다.

RegisterPage.tsx

역할: 사용자가 회원가입을 진행하는 페이지의 전체 UI를 구성합니다.

기능: useRegister 훅과 RegisterForm 컴포넌트를 연결하여 회원가입 폼의 입력 처리, 유효성 검사, 제출 기능을 구현합니다.

특징: LoginPage와 마찬가지로 UI와 로직이 명확히 분리된 구조를 가집니다.

MainPage.tsx

역할: 로그인 후 사용자가 가장 먼저 보게 되는 메인 페이지입니다.

기능: useRequireAuth 훅을 통해 인증된 사용자만 접근할 수 있도록 보호합니다. MainLayout과 HeroSection 등의 컴포넌트를 조립하고, useMainPage 훅을 통해 메뉴 클릭 등 상호작용을 처리합니다.

특징: 인증이 필요한 핵심 콘텐츠 페이지 역할을 합니다.

MyPage.tsx

역할: 사용자의 개인 정보, 프로젝트, 일정 등을 종합적으로 보여주는 마이페이지입니다.

기능: useMyPage 훅으로부터 사용자 정보, 캘린더 데이터 및 관련 함수들을 받아와 ProfileSection, CalendarSection 등 여러 컴포넌트에 전달합니다.

특징: 데이터 로딩 상태를 처리하며, 사용자의 개인화된 정보를 보여주는 대시보드 역할을 합니다.

OAuthSuccessPage.tsx

역할: 소셜 로그인 성공 후, 최종 서버 인증이 처리되는 동안 사용자에게 현재 상태를 보여주는 리다이렉트 페이지입니다.

기능: useOAuthSuccess 훅의 상태(loading, success, error)에 따라 다른 메시지를 ProcessingStatus 컴포넌트에 표시합니다.

특징: 사용자가 직접 조작하기보다는, 자동화된 인증 흐름의 중간 과정과 피드백을 담당합니다.
