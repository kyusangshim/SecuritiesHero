# 전역적으로 사용되는 TypeScript 타입 정의📁 src/types/
역할: 애플리케이션 전체에서 사용되는 TypeScript의 타입(Type)과 인터페이스(Interface)를 중앙에서 정의하고 관리합니다. 이를 통해 데이터의 형태를 일관되게 유지하고 코드의 안정성을 높입니다.

포함된 파일 및 역할:

auth.ts: 인증 및 사용자와 관련된 모든 타입을 정의합니다.

UserDto, LoginForm, RegisterData 등 인증 과정에 필요한 데이터 구조를 포함합니다.

calendar.ts: 캘린더 기능과 관련된 타입을 정의합니다.

CalendarEvent (개별 일정), Project (프로젝트) 등 캘린더에 표시될 데이터의 구조를 포함합니다.

index.ts: 모든 타입을 한 번에 내보내는(export) 역할을 합니다.

auth와 calendar의 모든 타입을 다시 내보내고, ApiResponse와 같이 여러 곳에서 쓰이는 공용 타입도 정의합니다.

특징:

타입 안전성: 데이터의 구조를 미리 정의하여 개발 과정에서 발생할 수 있는 오류를 방지합니다.

중앙 관리: 모든 데이터 모델이 types 폴더에 모여 있어 관리가 쉽고, 전체 데이터 구조를 파악하기 용이합니다.

모듈화: 기능(도메인)별로 타입 파일을 분리(auth.ts, calendar.ts)하여 체계적으로 관리합니다.
